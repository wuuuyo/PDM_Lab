# PDM Lab 网站介绍与运营 SOP

## 1. 网站一句话介绍

PDM Lab 是面向产品人的知识库管理与速查空间，帮助用户沉淀产品知识、查询方法与术语，并通过学习主线和路径规划辅助新人建立产品能力框架。

## 2. 核心价值

- 把零散产品知识整理成结构化知识库。
- 把方法论、技术架构、业务词语、权限安全、工作流程等内容变成可查可学的知识点。
- 支持收藏、笔记、个人知识库和复盘，方便长期沉淀。
- 提供学习主线和 4 个目标计划，让新人、求职、实习、转行用户快速找到路径。
- 提供论坛、反馈和通知，形成轻量社区与产品迭代闭环。

## 3. 核心模块说明

### 首页
用于建立网站定位并提供高频入口。

主要入口：
- 知识库
- 知识速查
- 知识图谱
- 学习主线
- 4 个路径规划
- 工具库
- 论坛
- 我的空间

### 知识库
网站核心模块。用于浏览 8 个专题下的知识点，支持搜索、速查、图谱联动。

运营重点：
- 保证专题数量准确。
- 保证知识点无乱码。
- 保证卡片标题、摘要、正文结构清晰。
- 保证相关链接可跳转。

### 学习主线与路径规划
用于帮助用户从知识库中找到学习顺序。

结构：
- 学习主线：默认推荐。
- 新人计划：适合产品新人。
- 求职计划：适合准备面试和作品集。
- 实习计划：适合进入实习和业务协作。
- 转行计划：适合非产品背景转入。

运营重点：
- 路径按钮必须对应真实知识点。
- 学习主线和 4 个计划要语义区分。
- 不要让学习路径喧宾夺主。

### 我的空间
登录用户的个人沉淀区。

包含：
- 我的收藏
- 我的笔记
- 我的知识库
- 复盘记录

### 我的账号
独立于我的空间，用于账户管理。

包含：
- 邮箱
- 昵称
- 修改密码
- 退出登录

### 管理者后台
超级管理员使用。

包含：
- 用户与角色
- 权限配置
- 知识管理
- 反馈管理
- 通知与发布相关能力

## 4. 内容维护 SOP

### 新增公开知识点

1. 在 `knowledge/product-knowledge-public` 中选择对应专题 Markdown。
2. 按统一结构新增内容：
   - 核心解释
   - 适用场景
   - 案例
   - PM 应用
   - 常见误区
   - 相关知识
3. 脱敏真实客户、公司和内部系统名称。
4. 运行导入脚本：

```powershell
node scripts\import-knowledge-base.mjs
node scripts\sync-knowledge.mjs
node scripts\write-config.mjs
```

5. 运行检查：

```powershell
node scripts\check-encoding.mjs
node --check site\data.js
```

6. 本地预览知识库和文章详情。

### 修改已有知识点

1. 先定位 Markdown 源文件，不直接改 `site/data.js`。
2. 修正文案、结构、案例或链接。
3. 重新生成知识数据。
4. 检查乱码和坏链接。
5. 验证知识速查、知识图谱、搜索是否还能命中。

### 删除知识点

1. 删除 Markdown 中对应条目。
2. 搜索全站是否还有链接指向该知识点。
3. 替换学习路径或相关知识中的旧链接。
4. 重新生成数据并运行坏链接检查。

## 5. UI 修改 SOP

### 修改首页

1. 确认是否影响网站定位。
2. 桌面端先调整信息层级。
3. 移动端重新检查搜索框、按钮、顶部栏。
4. 同步中英文文案。
5. 保持简洁大气，不新增过多卡片。

### 修改知识库

1. 不改变 8 个专题事实。
2. 搜索、知识速查、知识图谱保持同级。
3. 知识点卡片只展示必要信息。
4. 正文排版避免大段堆叠。

### 修改移动端

1. 先看是否溢出屏幕。
2. 再看点击区域是否被遮罩。
3. 长页面检查返回、回到顶部、滚动位置。
4. 按钮同一行排布时保证可点和不换行。

## 6. 权限与账号 SOP

### 未登录用户
- 可浏览公开知识库。
- 可看论坛帖子。
- 点击收藏、笔记、反馈提交等需要登录提示。

### 登录用户
- 可收藏、记笔记、写我的知识库、复盘、参与论坛。
- 头像菜单进入我的账号和消息通知。

### 超级管理员
- 可进入管理者后台。
- 可配置角色权限。
- 可复制内容。
- 可管理知识发布与反馈。

## 7. Supabase SOP

涉及数据库能力时：

1. 先确认前端提示的 SQL 文件。
2. 到 Supabase SQL Editor 执行。
3. 检查表、字段、RLS 策略是否存在。
4. 再回到本地预览功能。

常见 SQL：
- `supabase/schema.sql`
- `supabase/feedback.sql`
- `supabase/rbac.sql`
- `supabase/role-permissions.sql`
- `supabase/retention.sql`

## 8. 发布 SOP

### 发布前

```powershell
node scripts\check-encoding.mjs
node --check site\app.js
node --check site\data.js
git status --short
```

### 提交

```powershell
git add .
git commit -m "Update PDM Lab"
git push origin main
```

### 发布后

1. 查看 Cloudflare Pages 是否构建成功。
2. 强刷线上页面。
3. 验证首页、知识库、学习路径、文章详情、我的空间。
4. 如果线上没变化，检查缓存版本号和 Cloudflare 构建记录。

## 9. 日常运营节奏

### 每周
- 补充 3-5 个知识点。
- 检查反馈和论坛评论。
- 修复用户反馈的坏链接或移动端体验问题。

### 每月
- 整理新增知识进入专题结构。
- 检查 8 个专题是否平衡。
- 复盘首页入口是否还清晰。
- 检查 Supabase 数据和权限是否健康。

### 每次大改版
- 先写改版目标。
- 再调整信息架构。
- 最后做 UI 和文案。
- 发布前做全链路验收。
