# 公开版产品知识库 - 使用指南

> 5 分钟读懂:这套库是什么、怎么用、怎么学

---

## 1. 这是什么?

一套按"PM 成长路径"组织的通用产品知识库,覆盖:
- **方法论**:100+ 个(AARRR、KANO、MVP、RICE、北极星、RACI...)
- **行业术语**:300+ 个(ERP/CRM/IPD/SSO/RBAC/云-边-端/OTA...)
- **系统架构**:PM 视角的技术深度(10 个必会架构问题)
- **工作流程**:需求处理 7 步 + PRD 模板 + 复盘 SOP
- **每日学习**:130+ 个方法论(每天 3 个,持续推送)

## 2. 适用谁?

| 角色 | 价值 |
|------|------|
| **产品新人**(0-2 年) | 系统化补 PM 基本功 |
| **跨行业转岗 PM** | 快速理解 B 端 / 机器人 / 云服务术语 |
| **学生/求职** | 转正答辩 / 面试 / 简历方法论库 |
| **资深 PM** | 团队内训 / 跨部门沟通 / 评审会准备 |

## 3. 不适用谁?

- **5+ 年产品老兵**:本套库偏入门体系,深度不够
- **设计师/研发/运营**:术语字典可参考,方法论部分是 PM 视角
- **创业 CEO**:需要的是商业战略,本套库更偏执行

## 4. 怎么读?(3 种方式)

### 方式 1:碎片化(每天 3 分钟)
> 看 `06-每日学习/daily-log.md`,每次 1 个方法论
> 适合:忙碌的 PM 日常

### 方式 2:体系化(8 周)
> 按 `README.md` 4 阶段学习路径走
> 适合:想搭体系的产品新人

### 方式 3:应急速查(1-30 分钟)
> 用 `07-快速参考/关键词速查表.md` 定位
> 适合:写 PRD 遇到术语 / 评审会被挑战

## 5. 推荐阅读顺序(新人)

```
第 1 周   → 01-PM方法论/产品经理八股.md (通读 30 分钟)
第 2 周   → 06-每日学习/daily-log.md (看 30 个案例,每天 5 个)
第 3 周   → 01-PM方法论/产品策划方法论.md (战略/用户/需求 3 大体系)
第 4 周   → 01-PM方法论/产品策划方法论.md (设计/数据/协作 3 大体系)
第 5 周   → 05-工作流程/工作流程.md (7 步流程 + PRD 模板)
第 6 周   → 02-技术架构/系统架构.md (PM 必会 10 个架构问题)
第 7 周   → 02-技术架构/行业通用词语.md (技术/业务/权限术语)
第 8 周   → 实战:用学到的方法论 owner 一个中型需求
```

## 6. 速查命令(类 Unix)

```bash
# 找某个方法论
grep -r "RICE" 01-PM方法论/

# 看术语
grep -A 2 "^### " 02-技术架构/行业通用词语.md

# 看最近 7 天推送
tail -100 06-每日学习/daily-log.md
```
